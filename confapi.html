Configuring The API Server

Now that you have configured everything else, the last step is to put the finishing touches on the API server. The config file for which looks like this:

apitoolkit:
    apiName: api
    attempts: 5
    procCores: 8
    roles: ['ROLE_USER','ROLE_ROOT','ROLE_ADMIN','ROLE_ARCH']
    views: ['admin','apidoc','api','login','logout']
    apiServer: ['http://localhost:8080']
    apichain:
        limit: 3
    rest:
        postcrement: false
    chaining:
        enabled: true
    batching:
        enabled: true
    encoding: UTF-8
    throttle:
        rateLimit: {'ROLE_USER':1000,'ROLE_ADMIN':10000}
        dataLimit: {'ROLE_USER':1000000,'ROLE_ADMIN':1000000}
        # in seconds (3600 = 60 minutes)
        expires: 3600
    user:
        roles: ['ROLE_USER']
    admin:
        roles: ['ROLE_ROOT','ROLE_ADMIN','ROLE_ARCH']
    serverType: master
    webhook:
        services: ['iostate']
    corsInterceptor:
        includeEnvironments: ['development','test','production']
        excludeEnvironments: []
        allowedOrigins: ['http://localhost','http://localhost:8080','http://127.0.0.1','http://localhost/','http://localhost:8080/','http://127.0.0.1/']

---
root:
    login: admin
    password: admin_password
    email: youremail@yourdomain.com

The Only Things You Have To Change...

Don't be intimidated! There are only a couple of things that have to be changed and configured here:

  • login/password/email: This login and password will be used in testing and will provide a default user for you to login to the application with.
